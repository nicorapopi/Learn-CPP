# C vs C++
C และ C++ เป็นสองภาษาโปรแกรมยอดนิยมที่มีความคล้ายคลึงกันบางประการ แต่ก็มีความแตกต่างที่สำคัญหลายอย่าง C++ นั้นพัฒนาต่อยอดมาจากภาษา C โดยเพิ่มคุณสมบัติใหม่ๆ เช่น การเขียนโปรแกรมเชิงวัตถุ (Object-Oriented Programming - OOP), คลาส (Classes) และการจัดการข้อผิดพลาด (Exception Handling) แม้ว่าทั้งสองภาษาจะถูกใช้สำหรับงานที่คล้ายกัน แต่พวกมันก็มีไวยากรณ์ (Syntax) และความหมาย (Semantics) ที่ต่างกัน ทำให้แต่ละภาษามีเอกลักษณ์เฉพาะตัว

## ไวยากรณ์และความหมาย (Syntax and Semantics)
### C
C เป็นภาษาโปรแกรมเชิงกระบวนการ (Procedural Programming Language)
- เน้นที่ฟังก์ชันและการเขียนโปรแกรมแบบมีโครงสร้าง (Structured Programming)
- ไม่รองรับวัตถุ (Objects) หรือคลาส (Classes)
- การจัดการหน่วยความจำต้องทำด้วยตัวเอง (Manual) โดยใช้ฟังก์ชันอย่าง `malloc` และ `free`

```c
#include <stdio.h>

void printHello() {
    printf("Hello, World!\n");
}

int main() {
    printHello();
    return 0;
}
```

### C++
C++ เป็นทั้งภาษาเชิงกระบวนการ (Procedural) และเชิงวัตถุ (Object-Oriented)
- รองรับทั้งฟังก์ชันและคลาส
- ผสมผสานรูปแบบการเขียนโปรแกรมที่หลากหลาย (Different Programming Paradigms)
- การจัดการหน่วยความจำสามารถทำด้วยตัวเอง (เหมือนภาษา C) หรือใช้ตัวสร้าง/ตัวทำลาย (Constructors/Destructors) และสมาร์ทพอยน์เตอร์ (Smart Pointers)

```cpp
#include <iostream>

class HelloWorld {
public:
    void printHello() {
        std::cout << "Hello, World!\n";
    }
};

int main() {
    HelloWorld obj;
    obj.printHello();
    return 0;
}
```

## การนำโค้ดกลับมาใช้ใหม่และความเป็นโมดูล (Code Reusability and Modularity)
### C
- การนำโค้ดกลับมาใช้ใหม่ทำได้ผ่านฟังก์ชันและการเขียนโปรแกรมเชิงโมดูล
- บรรลุความเชื่อมโยงสูง (High Cohesion) และความผูกพันต่ำ (Low Coupling) ผ่านการออกแบบเชิงโครงสร้าง
- สามารถสร้างคลังฟังก์ชัน (Function Libraries) และนำมาใช้ผ่านไฟล์ส่วนหัว (Headers)

### C++
- ให้ความสามารถในการนำโค้ดกลับมาใช้ใหม่ได้ดีกว่าด้วย คลาส, การสืบทอด (Inheritance) และการพหุสัณฐาน (Polymorphism)
- ความเป็นโมดูลของโค้ดได้รับการปรับปรุงผ่าน เนมสเปซ (Namespaces) และลำดับชั้นของวัตถุที่ออกแบบมาอย่างดี

## การจัดการข้อผิดพลาด (Error Handling)
### C
- การจัดการข้อผิดพลาดในภาษา C ส่วนใหญ่ทำผ่านรหัสส่งกลับ (Return Codes)
- ขาดการรองรับข้อยกเว้น (Exceptions) หรือกลไกการจัดการข้อผิดพลาดในตัวที่ซับซ้อน

### C++
- มีการจัดการข้อยกเว้น (Exception Handling) ซึ่งใช้จัดการข้อผิดพลาดที่อาจเกิดขึ้นระหว่างการทำงานของโปรแกรม
- ช่วยให้สามารถดักจับและจัดการข้อยกเว้นด้วยคำสำคัญ `try`, `catch`, และ `throw` ทำให้ควบคุมการจัดการข้อผิดพลาดได้ดียิ่งขึ้น

## บทสรุป
ทั้ง C และ C++ เป็นภาษาที่ทรงพลังและมีคุณสมบัติเฉพาะตัว ในขณะที่ C มีความเรียบง่ายและเน้นการเขียนโปรแกรมเชิงกระบวนการ C++ มอบความยืดหยุ่นในการใช้รูปแบบการเขียนโปรแกรมที่หลากหลายและการจัดระเบียบโค้ดที่ดีขึ้น การทำความเข้าใจความแตกต่างของทั้งสองภาษานี้จะช่วยให้คุณตัดสินใจได้ว่าภาษาใดเหมาะสมที่สุดสำหรับความต้องการและรูปแบบการลงมือทำของคุณ
